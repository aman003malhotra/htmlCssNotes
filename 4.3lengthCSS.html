Length of CSS

The Absolute Lengths

px


  • Pixels are perhaps best thought of as "device pixels" as this lenghts doesn't have to do anything
  • with the literal screen pixels in the display you are looking at
  • it's actually an angular measurement

in

  • inches are a physical measurement but in CSS land, they just map directly to pixels

cm & mm

  • 1 cm == 37.8 px
  • 1 mm == 0.1 cm == 3.78 px

The font-realtive length

em & rem

  • a relative unit
  • 1 em == parent font size == 16px
  • 1.2 em == 120 % parent font size
  • Although the length doesn't chnage when you change font-family, it does change when you change with font-size

Percentage

  • a length set in percentage is based on the length of same property of the parent element
  • for example, if an element renders at 450 px width, a child element with a width set to 50% ie 225px
  • 1.2 em == 120 % parent font size

ViewPort Width

  • 1 vw = 1% of the width of viewport
  • It is similar to percentage, except that the value remains consistent for all elements regardless of their parent elements or parent elements width
  • A bit like how rem units are always relative to the root

ViewPort Height

  • 1 vw = 1% of the height of viewport
  • It is similar to percentage, except that the value remains consistent for all elements regardless of their parent elements or parent elements width
  • A bit like how rem units are always relative to the root

Vmin & Vmax

  • this value will be whichever is smaller at the moment, vw or vh
  • In the standard use case of sizing type, this may be a more useful metric than vw or vh on their own in determining true screen size
  • Vmax value is whichever is the larger at the moment, vw or vh
